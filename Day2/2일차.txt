HTML 기본 구조

<img></img> 아니야

<img src="파일명" alt="이미지 대체 문자">
------ ---    -------  ---   -------------------
태그  속성  속성값 속성   속성값

예)
<img src=apple.jpg alt=사과> #이렇게 해도 오류는 안나지만 파일명에 띄어쓰기 있으면 오류나. 아래처럼 써야해.
<img src=맛있는 사과.jpg alt=사과> (X)
<img src="맛있는 사과.jpg" alt="사과"> (O)
<img src='맛있는 사과.jpg' alt='사과'> (O)

*속성의 값은 따옴표를 꼭 사용하지 않아도 되지만, 속성값에 띄어쓰기가 있을경우, 문제가 발생할 수 있으므로 따옴표를 쓰기를 권장 (따옴표, 쌍따옴표 둘다가능)

<!DOCTYPE html> : HTML문서의 선언이며, 버전5를 나타냅니다.
<html lang="en"> : lang 속성은 웹 접근성에 관한 명시입니다. 
		스크린 리더에서 언어를 인식하여 자동으로 음성을 				변환하거나, 해당 언어에 적합한 발음을 제공할 수 있도록 				합니다.
		(ko)는 한글

*웹 접근성
웹 접근성은 장애를 가진 사람과 장애를 가지지 않은 사람 모두 웹사이트를 이용할 수 있게 하는 방식을 말합니다.

<meta charset="UTF-8"> : charset 속성은 문자를 인코딩시에 문자코드를 설정하는 역할을 지정. 한국어, 영어, 일본어만 사용하는 경우 윈도우 기본문자셋인 euc-kr 문자셋을 사용해도 되지만, 다국어로 인코딩하기위해선 utf-8 문자셋을 사용하기 권장함.

<meta name="viewport" content="width=device-width, initial-scale=1.0">: 모바일 기기에서 화면을 어떻게 구성할것인지를 설정함.

서식 태그

<strong>
텍스트를 굵게 표현하고 싶을때 사용. <b>태그와 표현은 동일하지만 <strong> 태그는 실제로 페이지 내에 중요한 부분을 브라우저에게 알려주고 해석할때, 브라우저에서 지원되는 웹 접근성으로 인식됩니다. 따라서 스크린 리더를 통해 페이지를 해석할때, 거센 억양으로 음을 낼수있도록 강조.

<em>
텍스트를 이탤릭체로 표현하고 싶을때 사용. <i>태그와 표현은 동일하지만 <em>태그는 텍스트를 이탤릭체로 변환하고 그 내용이 중요하다는 의미도 함께 포함.

<mark>
텍스트에 하이라이팅 효과를 적용시켜줍니다.

<del>
텍스트 중앙에 가로줄을 만들어 텍스트를 지운것과 같은 효과를 줍니다.

<ins>
텍스트 밑에 선을 그어 텍스트의 강조효과를 줍니다.

<sup>,<sub>
<sup>(superscript) 태그는 위첨자에 사용하고, <sub>(subscript) 태그는 아래첨자를 표현할때 사용함.
